## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge padding, inline tight spacing |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, table cell padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, card body padding |
| lg | 24px | `p-6` / `gap-6` | Section padding, sidebar item vertical spacing |
| xl | 32px | `p-8` / `gap-8` | Layout column gaps, main content top padding |
| 2xl | 48px | `py-12` | Major section breaks, page hero areas |
| 3xl | 64px | `py-16` | Page-level vertical breathing room |

Exceptions:
- Sidebar width: fixed 240px (`w-60`) — not an 8pt multiple but a standard sidebar convention
- Touch / click target minimum: 44px height for all interactive controls (buttons, select, input) — applied via `min-h-[44px]`
- Progress bar height: 8px (`h-2`) — intentionally compact for inline status indicators

Source: default 8pt scale; exceptions determined by researcher discretion

## Typography

All type is rendered in the system UI stack. No external font load. UI language is Spanish (CONTEXT.md D-22).

| Role | Size | Tailwind Class | Weight | Weight Class | Line Height |
|------|------|----------------|--------|--------------|-------------|
| Body | 14px | `text-sm` | Regular (400) | `font-normal` | 1.5 (`leading-relaxed`) |
| Label | 12px | `text-xs` | Semibold (600) | `font-semibold` | 1.4 (`leading-snug`) |
| Heading | 20px | `text-xl` | Semibold (600) | `font-semibold` | 1.2 (`leading-tight`) |
| Display | 28px | `text-3xl` | Semibold (600) | `font-semibold` | 1.1 (`leading-none`) |

**Rules:**
- Exactly 2 weights: Regular (400) for body text, Semibold (600) for labels, headings, buttons, and KPI numbers.
- Do not use font sizes outside this scale. The 4 sizes above cover all UI states.
- Table column headers: `text-xs font-semibold uppercase tracking-wide` (Label role + uppercase transform).

Source: researcher default; aligns with typical admin UI density for Spanish-language business tools

## Color

Tailwind CSS v4 CDN does not require a config file. Use Tailwind's built-in color palette. All values below reference Tailwind semantic names.

| Role | Tailwind Value | Hex Approximate | Usage |
|------|---------------|-----------------|-------|
| Dominant (60%) | `bg-white` / `bg-gray-50` | #ffffff / #f9fafb | Page background (`bg-gray-50`), main content area (`bg-white`) |
| Secondary (30%) | `bg-gray-100` / `bg-gray-800` | #f3f4f6 / #1f2937 | Sidebar (`bg-gray-800`), table header rows (`bg-gray-100`), cards |
| Accent (10%) | `bg-blue-600` / `text-blue-600` | #2563eb | Reserved for: primary CTA buttons, active sidebar item indicator, progress bar fill, "Load more" link |
| Destructive | `bg-red-600` / `text-red-600` | #dc2626 | Reserved for: failed job status badge, validation warning count badge, explicit destructive confirmation buttons |

**Semantic status colors (badges only):**
| Status | Color | Tailwind |
|--------|-------|---------|
| Completado | Green | `bg-green-100 text-green-800` |
| Procesando | Blue | `bg-blue-100 text-blue-800` |
| Pendiente | Gray | `bg-gray-100 text-gray-700` |
| Fallido | Red | `bg-red-100 text-red-800` |
| Con advertencias | Yellow | `bg-yellow-100 text-yellow-800` |

Accent reserved for: primary CTA buttons, active sidebar nav item left-border indicator, batch progress bar fill, "Cargar mas" trigger link. Never use accent for hover states, secondary actions, or informational text.

## Layout

**Shell (base.html):**
- Full viewport height: `min-h-screen flex`
- Sidebar: `w-60 bg-gray-800 text-gray-100 flex-shrink-0` — always visible (CONTEXT.md D-19), no collapse behavior in Phase 14
- Main content area: `flex-1 bg-gray-50 overflow-y-auto`
- Main content inner wrapper: `max-w-7xl mx-auto px-8 py-8`

**Sidebar structure:**
```
[App name / logo]        → top of sidebar, 24px padding
[Nav items × 5]          → stacked links, 44px min-height each
  Dashboard
  Subir PDFs
  Polizas
  Historial de Lotes
  —                      → spacer
[Version indicator]      → bottom of sidebar, 12px text
```

Active sidebar item: left border `border-l-2 border-blue-400` + `bg-gray-700 text-white`. Inactive: `text-gray-300 hover:bg-gray-700 hover:text-white`.

**Page grid patterns:**
- Dashboard: 3-column stat card row (`grid grid-cols-3 gap-6`) + full-width records-needing-review table below
- Upload: single centered column (`max-w-2xl mx-auto`) — drop zone + progress + summary table stacked
- Poliza list: full width — summary bar above table, collapsible filter panel slides in from top
- Poliza detail: single column, field groups in `<dl>` definition list with 2-column label/value layout
- Job history: full width table

Source: CONTEXT.md D-15, D-19, D-20; RESEARCH.md Architecture Patterns

## Interaction Contracts

### HTMX Polling (Batch Progress)
- Endpoint: `GET /ui/batch/{batch_id}/status`
- Poll interval: `hx-trigger="every 2s"` (RESEARCH.md recommendation)
- Target: `#batch-status` div
- Terminal condition: when job is complete or failed, server returns `HX-Trigger: jobDone` header + full `batch_summary.html` partial
- The polling div `hx-swap="outerHTML"` replaces itself with the summary table upon terminal response

### Free-Text Search (Poliza List)
- Implementation: server-side DB LIKE query (RESEARCH.md: consistent with HTMX architecture)
- Trigger: `hx-trigger="keyup delay:300ms changed"` on search input
- Target: `#poliza-rows` tbody
- Endpoint: `GET /ui/polizas/rows?q={term}&aseguradora={val}&...`

### Load More (Poliza List)
- Batch size: 25 records per load
- Trigger: click on "Cargar mas" button at bottom of table
- HTMX: `hx-get="/ui/polizas/rows?offset={n}"` + `hx-swap="beforeend"` on tbody
- Button hides itself when server returns fewer than 25 rows (include `hx-swap-oob` on button element)

### Filter Panel Toggle
- Toggle button: `hx-on:click="document.getElementById('filter-panel').classList.toggle('hidden')"` — pure DOM toggle, no server round-trip needed

### Date Range Selector (Dashboard)
- Three preset buttons: "Ultimos 7 dias" | "Ultimos 30 dias" | "Rango personalizado"
- Custom range: two `<input type="date">` fields, shown only when custom is selected
- On selection: HTMX `hx-get="/ui/dashboard/stats"` + `hx-target="#dashboard-stats"` replaces stat cards

### Export Downloads
- From batch summary: `<a href="/ui/batch/{id}/export?format=xlsx">` — plain anchor, no HTMX needed
- From poliza detail: `<a href="/polizas/{id}/export?format=xlsx">` — plain anchor

Source: CONTEXT.md D-01 through D-18; RESEARCH.md Pattern 2; researcher discretion for specifics

## Accessibility Baseline

- All form inputs have associated `<label>` elements (not placeholder-only)
- Interactive elements have `min-h-[44px]` (touch/click target size)
- Status badges include visually-hidden text when color is the sole meaning: `<span class="sr-only">Estado:</span>`
- Focus rings preserved: never add `outline-none` without a visible `focus:ring` replacement
- Table headers use `<th scope="col">` for screen reader column association
- Drop zone: `<input type="file" multiple accept=".pdf">` inside `<label>` — keyboard and screen reader operable without JS drag-drop

Source: researcher defaults — WCAG 2.1 AA minimum for internal business tools
