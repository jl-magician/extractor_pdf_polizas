## Spacing Scale

Declared values (must be multiples of 4). All values map to established Tailwind utility classes already in use.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `px-1` | Icon gaps, field indicator dot offset |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing, input padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing, fields-pane padding |
| lg | 24px | `gap-6`, `p-6` | Section card padding (matches poliza_detail.html) |
| xl | 32px | `gap-8`, `p-8` | Page-level gaps (matches base.html main padding) |
| 2xl | 48px | `py-12` | Major section breaks |
| 3xl | 64px | not used | Reserved, not used in this phase |

Exceptions:
- Draggable divider: `w-1` (4px wide) — not a spacing token, it is a structural element width.
- Touch targets for nav back-button and "Revisar" button: `min-h-[44px]` — matches established nav item height in base.html.
- Editable input vertical padding: `py-0.5` (2px) — intentionally compact to match surrounding text rhythm.

Source: `base.html` uses `px-8 py-8` for main content; `poliza_detail.html` uses `p-6` for cards, `gap-4` for grid.

### Field Row Component (field_row.html)

Two modes: display and input. Toggled by JavaScript `onclick`. Saved by HTMX PATCH on blur.

**Display mode (default):**
```
<div id="field-{field_path_dashed}"
     class="flex items-center gap-2 py-1 [border-l-2 border-blue-400 pl-2 if corrected]">
  <span class="text-xs font-semibold uppercase tracking-wide text-gray-500 w-40 flex-shrink-0">
    {label}
  </span>
  <span class="field-display text-sm text-gray-900 cursor-pointer hover:bg-blue-50 rounded px-1">
    {value or '-'}
  </span>
</div>
```

**Input mode (on click):**
```
<input
  class="field-input text-sm border border-blue-400 rounded px-2 py-0.5 focus:outline-none w-full"
  type="{text|number|date}"
  value="{current_value}"
  hx-patch="/ui/polizas/{id}/fields"
  hx-trigger="blur, keyup[key=='Enter']"
  hx-vals='{"field_path": "{field_path}"}'
  hx-target="#field-{field_path_dashed}"
  hx-swap="outerHTML"
/>
```

On successful PATCH: server returns updated `field_row.html` partial with `is_corrected=True`.
On failed PATCH: server returns same partial with `hx-swap-oob` error message below the input.

### Corrected Field Indicator (D-06)

When `is_corrected=True`, the field row gains `border-l-2 border-blue-400 pl-2` on its outermost `<div>`. No dot icon. Border only — non-intrusive.

### Correction History Panel (D-09)

Collapsible using native HTML `<details>`/`<summary>` element — zero JavaScript required.

```
<details class="mt-4 border-t border-gray-200 pt-4" id="correction-history">
  <summary class="text-sm font-semibold text-gray-700 cursor-pointer select-none">
    Historial de Correcciones ({count})
  </summary>
  <div class="mt-2 space-y-2">
    <!-- one row per correction, chronological descending -->
  </div>
</details>
```

History row format:
```
<div class="flex items-start gap-4 py-2 border-b border-gray-100 text-sm">
  <span class="text-gray-500 text-xs w-36 flex-shrink-0">{corrected_at formatted}</span>
  <span class="text-xs font-semibold uppercase text-gray-500 w-40 flex-shrink-0">{field_path}</span>
  <span class="text-gray-400 line-through">{old_value or '-'}</span>
  <span class="text-gray-900 mx-1">→</span>
  <span class="text-gray-900">{new_value or '-'}</span>
</div>
```

## Interaction States

All interactive elements must have exactly these states declared.

### Editable Field Row

| State | Visual |
|-------|--------|
| Default (display) | `text-gray-900` value text, no border |
| Corrected (display) | `border-l-2 border-blue-400 pl-2` left border |
| Hover (display) | `hover:bg-blue-50 rounded px-1` on value span |
| Active (editing) | Display span hidden; input visible with `border border-blue-400` |
| Saving (after blur) | HTMX in-flight: no spinner required; input stays visible until response |
| Saved | Server returns updated partial — field-row re-renders with corrected indicator |
| Error (PATCH failed) | Server returns partial with inline error text below input in `text-red-600 text-xs` |

### Draggable Divider

| State | Visual |
|-------|--------|
| Default | `w-1 bg-gray-300` |
| Hover | `hover:bg-blue-400` |
| Active (dragging) | `bg-blue-400` (JS adds class on mousedown, removes on mouseup) |

|-------|--------|

## Implementation Notes for Executor

1. `poliza_review.html` must NOT use `{% extends "base.html" %}`. It is a standalone document. Include CDN tags directly (Tailwind @4 and HTMX 2.0.8 — same versions as base.html).

2. Field type mapping for `<input type="">`: use `type="number"` for `prima_total`, `primer_pago`, `subsecuentes`, `suma_asegurada`, `deducible`; `type="date"` for `inicio_vigencia`, `fin_vigencia`, `fecha_emision`, `fecha_nacimiento`; `type="text"` for all other fields including `campos_adicionales` keys.

3. The PATCH endpoint must handle all three field_path namespaces: `"prima_total"` (top-level poliza column), `"asegurados.{id}.nombre_descripcion"` (nested ORM row by ID), `"campos_adicionales.{key}"` (JSON dict key). Dot-notation defined in D-08.

4. Dual-write (D-07): PATCH must write to both `polizas`/`asegurados`/`coberturas` AND `corrections` table in a single transaction. Both writes committed together or neither.

5. The draggable divider JS must clamp `newLeft` to `[200, containerWidth - 200]` to prevent panes from collapsing to zero width.

6. HTMX `hx-trigger="blur, keyup[key=='Enter']"` on the input: on Enter, the input loses focus naturally, which also fires blur — the PATCH fires once, not twice, because HTMX deduplicates within the same request cycle.
