       **App setup:**
       ```python
       from typing import Annotated, Generator, Optional
       from datetime import date
       from fastapi import Depends, FastAPI, HTTPException, Query
       from fastapi.responses import JSONResponse
       from sqlalchemy import select
       from sqlalchemy.orm import Session, selectinload

       ```

       **Startup event:**
       ```python
       @app.on_event("startup")
       def on_startup() -> None:
           engine = init_db(settings.DB_PATH)
           SessionLocal.configure(bind=engine)
       ```

       **DB dependency:**
       ```python
       def get_db() -> Generator[Session, None, None]:
           db = SessionLocal()
           try:
               yield db
           finally:
               db.close()

       DbDep = Annotated[Session, Depends(get_db)]
       ```

       **Routes:**

       `GET /polizas` — list with filters and pagination:
       - Query params: aseguradora (Optional[str]), tipo_seguro (Optional[str]), nombre_agente (Optional[str]), desde (Optional[date]), hasta (Optional[date]), skip (int=0, ge=0), limit (int=50, ge=1, le=500)
       - Build `select(Poliza)` with `selectinload(Poliza.asegurados)` and `selectinload(Poliza.coberturas)`
       - Append `.where()` for each non-None filter
       - For desde: `Poliza.inicio_vigencia >= desde`
       - For hasta: `Poliza.fin_vigencia <= hasta`
       - Apply `.offset(skip).limit(limit)`
       - Execute, convert each via `orm_to_schema(p).model_dump(mode="json")`
       - Return `JSONResponse(content=data)`

       `GET /polizas/{poliza_id}` — single policy:
       - Query with selectinload by id
       - If not found: `raise HTTPException(status_code=404, detail="Policy not found")`
       - Return `JSONResponse(content=orm_to_schema(poliza).model_dump(mode="json"))`

       `POST /polizas` — create:
       - Accept `PolicyExtraction` body (FastAPI auto-validates via Pydantic)
       - Call `upsert_policy(db, extraction)` — upsert handles create-or-update
       - Return `JSONResponse(content=orm_to_schema(poliza).model_dump(mode="json"), status_code=201)`

       `PUT /polizas/{poliza_id}` — update:
       - Verify poliza_id exists, 404 if not
       - Accept PolicyExtraction body
       - Load existing poliza, update its fields from extraction using the same pattern as upsert_policy but targeted to the specific ID
       - Simpler: just call `upsert_policy(db, extraction)` which handles update by (numero_poliza, aseguradora)
       - BUT the PUT is by ID, not by (numero_poliza, aseguradora). So: find poliza by ID first (404 if missing), then update scalar fields + children in-place using same logic as upsert_policy. OR: accept that PUT replaces the policy at that ID using the extraction data. Implementation: query by ID, if not found raise 404, then update all scalar fields from extraction body, clear+recreate children, commit. Return updated poliza via orm_to_schema.

       `DELETE /polizas/{poliza_id}` — delete:
       - Find by id, 404 if not found
       - `session.delete(poliza)` — cascade handles children
       - `session.commit()`
       - Return `JSONResponse(content={"detail": "Policy deleted"}, status_code=200)`
