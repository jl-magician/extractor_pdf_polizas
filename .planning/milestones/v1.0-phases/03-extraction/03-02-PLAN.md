1. Create `policy_extractor/extraction/client.py` with:

   - Import `anthropic`, `loguru.logger`, `pydantic.ValidationError`
   - Import `PolicyExtraction` from schemas, `SYSTEM_PROMPT_V1` and `PROMPT_VERSION_V1` from prompt, `build_extraction_tool` and `TOOL_NAME` from schema_builder, `settings` from config
   - `call_extraction_api(client: anthropic.Anthropic, assembled_text: str, model: str, max_tokens: int = 4096) -> anthropic.types.Message`:
     - Calls `client.messages.create()` with:
       - `model=model`
       - `max_tokens=max_tokens`
       - `system=SYSTEM_PROMPT_V1`
       - `messages=[{"role": "user", "content": assembled_text}]`
       - `tools=[build_extraction_tool()]`
       - `tool_choice={"type": "tool", "name": TOOL_NAME}`
     - Returns the raw Message object
   - `parse_and_validate(message, ingestion_file_hash: str) -> tuple[PolicyExtraction, dict]`:
     - Asserts `message.content[0].type == "tool_use"` — if not, raises ValueError with descriptive message
     - Extracts `raw_input: dict = message.content[0].input`
     - Injects provenance fields into raw_input:
       - `raw_input["source_file_hash"] = ingestion_file_hash`
       - `raw_input["model_id"] = message.model`
       - `raw_input["prompt_version"] = PROMPT_VERSION_V1`
       - `raw_input["extracted_at"] = datetime.now(timezone.utc)`
     - Creates `PolicyExtraction(**raw_input)` — may raise ValidationError
     - Returns `(policy, raw_input)` where raw_input is the original dict (before provenance injection) for audit
   - `extract_with_retry(client: anthropic.Anthropic, assembled_text: str, ingestion_file_hash: str, model: str, max_retries: int = 1) -> tuple[PolicyExtraction, dict] | None`:
     - Calls `call_extraction_api` and `parse_and_validate`
     - On `ValidationError`: log the error, if retries remaining, append validation error to assembled_text as a correction prompt:
       ```
       f"{assembled_text}\n\nIMPORTANT: Your previous response failed validation:\n{exc}\nPlease correct the fields and respond again."
       ```
       Then retry.
     - On second failure or any other exception: log error, return None
     - On success: return `(policy, raw_response_dict)`

2. Create `policy_extractor/extraction/verification.py` with:

   - `verify_no_hallucination(policy: PolicyExtraction, source_text: str) -> PolicyExtraction`:
     - Makes a copy of `policy.confianza` as a mutable dict
     - For each of these key fields: `("numero_poliza", policy.numero_poliza)`, `("aseguradora", policy.aseguradora)`
       - If the value is not None and `value.lower() not in source_text.lower()`:
         - Set `confianza[field_name] = "low"` (override whatever Claude reported)
         - Log a warning: `"Possible hallucination: {field_name}='{value}' not found in source text"`
     - Return `policy.model_copy(update={"confianza": confianza})`
  </action>
