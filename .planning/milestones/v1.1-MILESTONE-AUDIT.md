## Cross-Phase Integration

### Latent Fragilities (non-breaking)

1. **SessionLocal binding order dependency** — Both `_process_single_pdf()` (Phase 9) and `_run_extraction()` (Phase 8) call `SessionLocal()` without calling `_setup_db()`. They rely on the parent entry point having called it first. Works in production but fragile if functions are reused from alternate entry points.

2. **WAL mode is engine-level, not session-level** — WAL is set once via `PRAGMA journal_mode=WAL` in `get_engine()`. This is correct for SQLite (WAL is a database-level property, not connection-level), but the pattern relies on the engine being created before any worker sessions.
