         - Deep copies data via `copy.deepcopy`
         - If "campos_adicionales" is a dict at top level, pop each key in SKIP_CAMPOS_KEYS

    3. Create `policy_extractor/regression/field_differ.py`:
       - `SKIP_FIELDS = frozenset({"confianza", "source_file_hash", "model_id", "prompt_version", "extracted_at", "_source_pdf"})`
       - `LIST_MATCH_KEYS = {"asegurados": "nombre_descripcion", "coberturas": "nombre_cobertura"}`
       - Dataclass `DriftReport`:
         - `rows: list[tuple[str, str, str, str]]` — (field, expected, actual, status)
         - Property `has_failures` — True if any row has status "FAIL"
         - Method `format_table() -> str` — returns plain text with header "Field | Expected | Actual | Status" and separator line, then one row per drift entry. Truncate values to 50 chars for readability.
       - Class `FieldDiffer`:
         - `__init__(self, expected: dict, actual: dict)` stores both
         - `compare() -> DriftReport`:
           a. Iterate expected top-level keys. Skip if key in SKIP_FIELDS. Skip if value is "[REDACTED]".
           b. For scalar keys (not in LIST_MATCH_KEYS, not "campos_adicionales"): compare `expected[k] == actual.get(k)`. If not equal, add FAIL row.
           c. For "campos_adicionales": for each key in expected["campos_adicionales"], check key present in actual["campos_adicionales"] and value equal. Missing key = FAIL. Extra keys in actual = ignored (acceptable improvement).
           d. For "asegurados": match expected items to actual items by "nombre_descripcion" value (skip if "[REDACTED]"). For each matched pair, compare non-REDACTED, non-SKIP fields. Unmatched expected items = FAIL. Count mismatch (actual has fewer) = FAIL.
           e. For "coberturas": same logic as asegurados but match by "nombre_cobertura". Compare all non-REDACTED fields within matched pairs.
           f. Return DriftReport with all collected rows.
